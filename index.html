Trilha de Aprendizado SQL

Do Zero ao SELECT: Fundamentos e Operação

Módulo 1

Infraestrutura e Conexão

Antes de escrever código, você precisa garantir que o "motor" (servidor) está funcionando.

  • Identificar Servidor: Use hostname no CMD para descobrir o nome da sua máquina local. É o endereço que você usará no campo "Server Name" do SSMS.
  • Nova Consulta: Atalho Ctrl + N. Lembre-se: cada aba é uma conexão independente. Se fechar e abrir, deve garantir que o banco correto está selecionado no dropdown superior esquerdo.
Módulo 2

Importação de Dados (Restore)

O processo de restaurar um banco (.bak) reconstrói toda a estrutura de tabelas e registros.

Direito em Databases > Restore Database > Device > Add > [Caminho do Arquivo] > OK.

Dica de Professor: Após restaurar, se o banco não aparecer, clique com o botão direito na pasta "Databases" e escolha Refresh.

Módulo 3

Consultas de Dados (DQL)

Abaixo, os comandos fundamentais para extrair inteligência dos dados:

1. SELECT e FROM

O SELECT define o que você quer ver. O FROM define de onde vem.

-- Boa prática: Identação para leitura rápida
SELECT
    NomeProduto,
    PrecoUnitario
FROM
    DimProduct;

2. DISTINCT

Remove duplicatas da visualização. Útil para descobrir categorias únicas em uma coluna de milhares de linhas.

SELECT DISTINCT ColorName FROM DimProduct;

3. TOP e PERCENT

Limita o volume de dados. Essencial para testar códigos em bases gigantescas sem travar o servidor.

SELECT TOP (10) * FROM Vendas; -- 10 primeiras linhas
SELECT TOP (5) PERCENT * FROM Vendas; -- 5% da base

4. ALIAS (AS)

Renomeia colunas para o resultado final (o "Output"). Não altera o nome na tabela original, apenas na sua visualização.

SELECT ColorName AS "Cor do Produto" FROM DimProduct;
Dica de Carreira

Regras de Ouro do Desenvolvedor SQL

  • Comentários (--): Documente o porquê de filtros complexos. No futuro, você agradecerá.
  • Visualização Rápida: O comando "Selecionar 1000 linhas superiores" é um atalho automático que o SSMS cria para você ganhar tempo.
  • Barra de Status: Sempre olhe o canto inferior direito. Se uma consulta demorar muito, você saberá quantos segundos passaram e se o volume de linhas está correto.
