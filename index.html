Importando uma Base de Dados

Para adicionar um banco de dados no SQL Server Management Studio (SSMS):

  1. Clique com o botão direito em Bancos de Dados.
  2. Selecione Restaurar Banco de Dados.
  3. Escolha a opção Dispositivo.
  4. Clique em Adicionar.
  5. Selecione o arquivo do banco.
  6. Clique em OK até finalizar.
  7. Aguarde a mensagem: Restaurado com êxito.
Após restaurar o banco, ele aparecerá na lista de bancos de dados.

Visualizando tabelas

Clique no símbolo + do banco → Tabelas.

Visualizando dados da tabela

Botão direito na tabela → Selecionar 1000 linhas superiores.

Descobrindo o nome do servidor

hostname

Execute no CMD para descobrir o nome do computador.

Nova janela de consulta

  • Botão direito no banco → Nova Consulta
  • Ou use o atalho: CTRL + N

Fundamentos do SELECT

SELECT e FROM

O comando SELECT serve para selecionar dados.

O comando FROM informa de qual tabela os dados serão buscados.

SELECT *
FROM DimProduct

Usando SELECT *

O asterisco (*) traz todas as colunas da tabela.

SELECT *
FROM DimCustomer

Selecionando colunas específicas

SELECT FirstName, EmailAddress, BirthDate
FROM DimCustomer
Boa prática: evite usar SELECT * em sistemas grandes. Busque apenas as colunas necessárias.

Usando AS para renomear colunas

O comando AS cria apelidos para colunas.

SELECT
    FirstName AS Nome,
    BirthDate AS DataNascimento,
    EmailAddress AS Email
FROM DimCustomer

SELECT DISTINCT

Retorna apenas valores únicos.

SELECT DISTINCT ColorName
FROM DimProduct

SELECT TOP

Limita a quantidade de linhas retornadas.

SELECT TOP (10) *
FROM DimProduct

TOP PERCENT

Traz uma porcentagem das linhas.

SELECT TOP (20) PERCENT *
FROM DimCustomer

Comentários no SQL

-- Isso é um comentário
SELECT * FROM DimProduct

Indentação

Indentar significa organizar o código para melhorar a leitura.

SELECT
    ProductName,
    UnitPrice,
    ColorName
FROM
    DimProduct
WHERE
    UnitPrice > 100

Filtros e Ordenação

ORDER BY

Usado para ordenar resultados.

Comando Função
ASC Ordem crescente
DESC Ordem decrescente 
SELECT DISTINCT ProductKey
FROM FactSales
ORDER BY ProductKey DESC
O ORDER BY sempre fica no final da consulta.

WHERE

Usado para filtrar registros.

SELECT ProductName, UnitPrice
FROM DimProduct
WHERE UnitPrice >= 1000

Filtro de texto

SELECT *
FROM DimProduct
WHERE ColorName = 'Black'

Filtro de data

SELECT FirstName, BirthDate
FROM DimCustomer
WHERE BirthDate > '1970-12-31'
ORDER BY BirthDate

AND

Todas as condições devem ser verdadeiras.

SELECT *
FROM DimProduct
WHERE ColorName = 'Black'
AND BrandName = 'Contoso'

OR

Pelo menos uma condição deve ser verdadeira.

SELECT *
FROM DimProduct
WHERE ColorName = 'Black'
OR BrandName = 'Fabrikam'

NOT

Inverte a condição.

SELECT *
FROM DimProduct
WHERE NOT ColorName = 'Black'

IN

Substitui vários OR.

SELECT *
FROM DimProduct
WHERE BrandName IN ('Contoso', 'Litware', 'Fabrikam')

LIKE

Usado para procurar textos parecidos.

Exemplo Significado
%texto% Contém texto
texto% Começa com
%texto Termina com 
SELECT *
FROM DimProduct
WHERE ProductName LIKE '%2G%'

BETWEEN

Filtra intervalos.

SELECT ProductName, UnitPrice
FROM DimProduct
WHERE UnitPrice BETWEEN 50 AND 100

NOT BETWEEN

SELECT ProductName, UnitPrice
FROM DimProduct
WHERE UnitPrice NOT BETWEEN 50 AND 100

IS NULL

Busca valores nulos.

SELECT *
FROM DimCustomer
WHERE CompanyName IS NULL

IS NOT NULL

SELECT *
FROM DimStore
WHERE CloseDate IS NOT NULL

Funções de Agregação

As funções de agregação resumem dados.

SUM

Soma os valores de uma coluna.

SELECT
    SUM(SalesQuantity) AS QuantidadeVendida,
    SUM(ReturnQuantity) AS QuantidadeRetorno
FROM FactSales

COUNT

Conta registros.

SELECT COUNT(*)
FROM DimProduct

Contando apenas uma coluna:

SELECT COUNT(ProductDescription)
FROM DimProduct
COUNT(coluna) não conta valores nulos.

COUNT DISTINCT

Conta valores únicos.

SELECT COUNT(DISTINCT ColorName)
FROM DimProduct

MAX

Retorna o maior valor.

SELECT MAX(UnitPrice)
FROM DimProduct

MIN

Retorna o menor valor.

SELECT MIN(UnitPrice)
FROM DimProduct

AVG

Calcula a média.

SELECT AVG(UnitPrice)
FROM DimProduct

GROUP BY e HAVING

GROUP BY

O GROUP BY agrupa registros iguais.

SELECT
    Gender,
    COUNT(*) AS Total
FROM DimEmployee
GROUP BY Gender

GROUP BY + ORDER BY

SELECT
    BrandName,
    COUNT(*) AS Quantidade
FROM DimProduct
GROUP BY BrandName
ORDER BY Quantidade DESC

GROUP BY + WHERE

O WHERE filtra antes do agrupamento.

SELECT
    ColorName,
    COUNT(*) AS Total
FROM DimProduct
WHERE BrandName = 'Contoso'
GROUP BY ColorName

HAVING

O HAVING filtra depois do agrupamento.

SELECT
    BrandName,
    COUNT(*) AS Quantidade
FROM DimProduct
GROUP BY BrandName
HAVING COUNT(*) > 100

Diferença entre WHERE e HAVING

WHERE HAVING
Filtra linhas antes do GROUP BY Filtra grupos depois do GROUP BY
Não usa funções agregadas Usa funções agregadas

Exemplo prático

SELECT
    BrandName,
    AVG(UnitPrice) AS MediaPreco
FROM DimProduct
WHERE UnitPrice > 100
GROUP BY BrandName
HAVING AVG(UnitPrice) > 500
ORDER BY MediaPreco DESC

Boas Práticas em SQL

  • Use indentação para melhorar a leitura.
  • Evite SELECT * em ambientes profissionais.
  • Use aliases com AS para deixar consultas claras.
  • Comente consultas importantes usando --.
  • Organize cláusulas na ordem correta:
SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY

Exemplo profissional

-- Produtos acima de 1000 reais
SELECT
    ProductName AS Produto,
    BrandName AS Marca,
    UnitPrice AS Preco
FROM
    DimProduct
WHERE
    UnitPrice >= 1000
    AND ColorName = 'Black'
ORDER BY
    UnitPrice DESC
Quanto mais organizado estiver seu SQL, mais fácil será manutenção e análise.

Mais Exemplos Práticos de SQL

SELECT com cálculos

Também podemos fazer cálculos diretamente nas consultas SQL.

SELECT
    ProductName,
    UnitPrice,
    UnitPrice * 2 AS PrecoDobrado
FROM DimProduct

Nesse exemplo o SQL cria uma nova coluna calculando o dobro do preço.

ORDER BY com múltiplas colunas

Podemos ordenar por mais de uma coluna.

SELECT
    ProductName,
    BrandName,
    UnitPrice
FROM DimProduct
ORDER BY BrandName ASC, UnitPrice DESC

Primeiro ele organiza pela marca e depois pelo preço.

Usando parênteses no WHERE

Os parênteses ajudam a definir prioridade nas condições.

SELECT *
FROM DimProduct
WHERE
(
    BrandName = 'Contoso'
    OR BrandName = 'Fabrikam'
)
AND UnitPrice > 100

Sem os parênteses o resultado pode ser diferente.

LIKE avançado

Buscando produtos que começam com TV:

SELECT *
FROM DimProduct
WHERE ProductName LIKE 'TV%'

Buscando produtos que terminam com HD:

SELECT *
FROM DimProduct
WHERE ProductName LIKE '%HD'

COUNT com GROUP BY

Conta quantos produtos existem por marca.

SELECT
    BrandName,
    COUNT(*) AS QuantidadeProdutos
FROM DimProduct
GROUP BY BrandName
ORDER BY QuantidadeProdutos DESC

SUM com GROUP BY

Soma valores agrupados.

SELECT
    channelkey,
    SUM(SalesAmount) AS TotalVendas
FROM FactSales
GROUP BY channelkey

AVG com GROUP BY

Calcula média por grupo.

SELECT
    BrandName,
    AVG(UnitPrice) AS MediaPreco
FROM DimProduct
GROUP BY BrandName
ORDER BY MediaPreco DESC

MAX e MIN juntos

SELECT
    MAX(UnitPrice) AS MaiorPreco,
    MIN(UnitPrice) AS MenorPreco
FROM DimProduct

IS NULL e IS NOT NULL na prática

Descobrindo lojas fechadas:

SELECT StoreName, CloseDate
FROM DimStore
WHERE CloseDate IS NOT NULL

Descobrindo lojas abertas:

SELECT StoreName
FROM DimStore
WHERE CloseDate IS NULL

IN substituindo vários OR

SELECT *
FROM DimProduct
WHERE ColorName IN ('Black', 'Silver', 'Red')

Isso evita escrever vários OR.

Exemplo profissional completo

-- Produtos caros da Contoso
SELECT
    ProductKey AS ID,
    ProductName AS Produto,
    BrandName AS Marca,
    ColorName AS Cor,
    UnitPrice AS Preco
FROM
    DimProduct
WHERE
    BrandName = 'Contoso'
    AND UnitPrice >= 1000
    AND ColorName IN ('Black', 'Silver')
ORDER BY
    UnitPrice DESC

Dicas importantes para iniciantes

  • Pratique consultas todos os dias.
  • Tente entender o resultado antes de executar.
  • Use comentários para organizar consultas grandes.
  • Evite SELECT * em tabelas muito grandes.
  • Use aliases para deixar relatórios mais profissionais.
  • Aprenda primeiro SELECT, WHERE e ORDER BY antes de avançar.
  • GROUP BY é muito usado em relatórios e dashboards.
